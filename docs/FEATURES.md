# Weave 기능 명세

macOS 메뉴바 포트폴리오 트래커. 메뉴바에 핀 자산 시세를 상시 표시하고,
클릭하면 360×720 팝오버에서 포트폴리오 전체를 관리한다.

- 스택: SwiftUI(MenuBarExtra) + Swift Charts, macOS 14+
- 디자인: `design/mockups-v5.html` 기준 (Slate/Light 테마, 링 게이지, 로고 마커 차트)
- 데이터 소스: Binance / Naver / Yahoo Finance 3개 고정, 사용자는 심볼 검색만

---

## 1. 메뉴바

- 표시 대상: 선택된 자산들을 N초 간격으로 로테이션(기본 10초). 특정 자산 핀 고정 가능.
- 형식: `BTC $60,000 ▲1.23%`, 등락 방향에 따라 ▲/▼.
  옵션: 풀(이름+가격+등락률) / 컴팩트(심볼+등락률) / 가격만.
- 표시할 자산이 없으면 포트폴리오 총액을 표시.
- 프라이버시 모드가 켜져 있으면 금액은 마스킹하고 등락률만 표시.
- 클릭 → 팝오버 열림. 팝오버 밖 클릭 시 닫힘.

## 2. 홈 (팝오버 메인)

### 2.0 헤더
- 좌측 새로고침 · 중앙 타이틀 · 우측 프라이버시 토글(eye) + 설정(gear). 아이콘은 8장 매핑 참조.
- **프라이버시 모드**: 켜면 앱 전체에서 금액(가격·평가금·손익액)을 마스킹,
  등락률(%)은 유지. 메뉴바에도 적용(1장).

### 2.1 링 게이지 3개
- **Day**: 오늘 등락 기여를 자산색 세그먼트로 분할. 포트폴리오가 이득이면
  이득 기여 종목만, 손실이면 손실 기여 종목만 포함(반대 부호 종목 제외).
  세그먼트 길이 = 해당 종목 기여액 비율.
- **Return**(중앙, 크게): 총 수익률. 분할 규칙은 Day와 동일.
- **Assets**: 평가금 비중 도넛(전 종목 자산색), 중앙에 종목 수.
- 세그먼트는 갭 없이 이어짐(butt cap). hover 시 굵어지며 기여액/비중 툴팁.
- 채움 스케일: Day 링 ±2% = 풀링, Return 링 ±25% = 풀링, 초과 시 꽉 찬 상태 유지.
- 자산 색: 고정 팔레트 8색을 추가 순서대로 자동 할당, 자산 관리에서 수동 변경.
  Assets 도넛은 비중 상위 4개 + 나머지 "기타"(회색) 묶음.

### 2.2 총액 영역
- 포트폴리오 총 평가금(기준통화 환산) + 등락률 솔리드 배지.

### 2.3 Value History 차트
- 필터: **통합**(포트폴리오 가치 라인 + 그라디언트 필) / **자산별**(종목별
  정규화 멀티라인, 시작점 0% 기준). 기간: 1M / 3M / 6M / 1Y.
- 표시 구간: max(선택 기간 시작, 첫 매수일)부터. 첫 매수 이전 구간은 없음.
- 통합 모드: 매수 이벤트 위치에 **자산 로고 원형 마커**(자산색 링).
  - hover → 툴팁(수량@가격 · 날짜 · 현재 대비 %) + 세로 가이드라인.
  - 클릭 → 해당 자산 상세로 이동.
  - 자산별 모드에서는 마커 숨김.
- 구현: Swift Charts `LineMark`+`AreaMark`, 마커는 `PointMark`+`annotation`,
  hover는 `chartOverlay`+`ChartProxy`.

### 2.4 Assets 리스트
- 행: 로고 · 이름 · 보유수량/매수횟수 · 현재가 · 등락 솔리드 배지.
- 정렬: 평가금 내림차순 고정.
- 행 클릭 → 자산 상세. 컨텍스트 메뉴: 핀 지정 / 숨김 / 삭제.

### 2.5 Footer
- 좌측: `Weave 0.1.0 · Next refresh in 4m` (버전 + 다음 시세 갱신 카운트다운).
  업데이트가 있으면 `Weave 0.1.0 → 0.2.0 사용 가능` 뱃지로 전환, 클릭 시 설치.
- 우측: ✎ 아이콘(자산 관리 화면 이동). 텍스트 버튼 없이 아이콘만.
- 설정 진입은 홈 헤더의 ⚙ 아이콘(9장).

## 3. 자산 관리 & 검색 (✎)

홈 footer의 ✎로 진입. 상단 검색바 + 보유 자산 관리 리스트로 구성.

### 3.1 검색 & 추가
- 검색창 하나. 2자 이상 입력 + 300ms 디바운스로 라이브서치.
- 3소스 병렬 질의 후 통합 리스트. 각 행에 마켓 뱃지: 크립토 / 국장 / 미장 / 일장 / 기타.
  - **Binance**: `exchangeInfo` 심볼 목록을 로컬 캐시 → 로컬 필터 (크립토, USDT 페어 기본)
  - **Naver**: `ac.stock.naver.com` 자동완성 (국장, 한글 검색)
  - **Yahoo**: `/v1/finance/search` (미장/일장/기타 글로벌, 폴백)
- 동일 종목이 여러 소스에서 나오면 하나만 표시: 국장=네이버, 크립토=Binance 우선.
  야후는 미장/일장 전담.
- 선택 시 provider · symbol · 통화 · 로고 자동 세팅으로 자산 생성. 중복 추가 방지.
- **Manual Asset**: 검색 불가 자산(부동산, 비상장 등)용. 이름/통화/평가액 수동 입력,
  시세 갱신 없음. 통합 차트에서 제외(또는 평단 고정선) 옵션.
- 로고: 크립토는 아이콘셋 번들, 주식은 로고 캐시 실패 시 이니셜 배지 폴백.

### 3.2 보유 자산 관리
- 자산 행: 로고 · 이름 · 메뉴바 표시 토글 · 색상 변경 · 숨김 · 삭제.
- 삭제는 거래 내역까지 통째 삭제, 확인 다이얼로그("거래 N건이 함께 삭제됩니다").
- 자산 수 soft limit 30개(API rate limit 보호).
- 빈 상태(자산 0개): 홈 대신 온보딩 화면 — 검색바 + "BTC, 삼성전자, AAPL 같은
  심볼을 검색해보세요" 안내.

## 4. 자산 상세

- 헤더: 뒤로 · 자산명 · 거래 추가(＋).
- 현재가 + 등락 배지 + 평단 대비 배지.
- **차트** (Swift Charts):
  - 실제 캔들 종가 라인 + 그라디언트 필(자산색), 우측 y축 가격, 하단 x축 날짜.
  - 평단 점선(`RuleMark` + dash) + 라벨.
  - 매수(B, 초록)/매도(S, 빨강) 원형 마커 — hover 툴팁, 위치는 실제 체결가 좌표.
  - hover 크로스헤어 + 날짜·가격 툴팁.
  - 기간: 1M / 3M / 6M / 1Y / ALL.
- **Trades 리스트**: 매수/매도 칩 · 수량@가격 · 날짜·메모 · 우측에
  매수는 "현재 대비 %", 매도는 "실현손익". 행 컨텍스트 메뉴로 수정/삭제.
- Trades 섹션 상단에 실현손익 누계 표시. 실현손익은 이 화면에만 노출
  (홈 링/총액은 미실현 기준 유지).

## 5. 거래 입력 (매수/매도)

- 필드: 종류(매수/매도) · 수량 · 단가 · 총액 · 날짜 · 메모.
- 수량/단가/총액 중 2개 입력하면 나머지 자동 계산.
- 과거 날짜 선택 시 그날 종가를 단가에 자동 프리필(캔들 캐시 활용, 수정 가능).
- 매도 검증: 보유 수량 초과 불가.
- 실현손익 = (매도단가 − 매도 시점 평단) × 수량. 평단은 이동평균법.

## 6. 시세 데이터 파이프라인

- **Quote(현재가)**: Binance ticker / Naver 시세 / Yahoo chart meta.
  주기 폴링(기본 300초, 설정 60~900초). 실패 시 마지막 값 유지 + stale 표시.
- **Candle(히스토리)**: Binance klines / Naver fchart 일봉 / Yahoo chart.
  일봉 기준, 로컬 캐시(하루 1회 갱신). 캐시 위치: `Application Support/Weave/cache/`.
  기간 매핑: 1M/3M/6M/1Y = 일봉, ALL = 주봉(5년 초과분은 월봉).
- **환율**: Yahoo `KRW=X`, `JPY=X` 등. 1시간 캐시. 합산 계산은 기준통화로 환산.
- 자산의 통화는 provider가 아니라 **상장 시장**이 정함(야후에서도 005930.KS는
  KRW). 자산 행 표시는 기본적으로 소스 통화 그대로, 환산은 합산에만 적용(9장).
- provider는 프로토콜로 추상화(Naver/Yahoo는 비공식 API — 스펙 변경 시 어댑터만 교체).

## 7. 포트폴리오 계산

- 종목: 평단(이동평균) · 보유수량 · 평가금 = 수량×현재가 · 미실현손익 · 수익률 ·
  일간 변동 · 비중.
- 일간 변동 기준은 소스 제공값 그대로: 크립토 = 24h 롤링(Binance), 주식 = 전일
  종가 대비. 억지로 통일하지 않음.
- 포트폴리오: 종목 합산(환산 후) · 총 수익률 · 일간 변동 · 링 기여도
  (포트폴리오 손익과 같은 부호인 종목의 손익 비율) · 실현손익 누계.
- **가치 시계열**(통합 차트용): 일별 Σ(그날 보유수량 × 그날 종가 × 그날 환율).
  보유수량은 거래 날짜 기준 스텝 함수. 캔들 캐시로 로컬 계산.

## 8. 테마 & 아이콘

- **Slate**: 푸른기 도는 차콜(#2B2D35 베이스) + 비비드 시스템 컬러.
- **Light**: 밝은 회색(#F4F4F6) + 라이트 시스템 컬러.
- 기본은 **시스템 외관 따라가기**(다크→Slate, 라이트→Light), 설정에서 수동 고정 가능.
- 색은 전부 시맨틱 토큰으로 관리(목업 변수 그대로 이식).

### 아이콘 정책
- 앱 내 모든 아이콘은 **SF Symbols** 사용. 이모지·컬러 글리프 금지.
- 모노크롬, 색은 시맨틱 토큰(보통 `--text2`, 활성 시 `--link`/`--green`)을 따름.
- HTML 목업의 글리프(↻, ⚙, ✎ 등)는 근사치일 뿐, 구현 기준은 아래 매핑.

| 위치 | 기능 | SF Symbol |
|------|------|-----------|
| 홈 헤더 좌 | 수동 새로고침 | `arrow.clockwise` |
| 홈 헤더 우 | 프라이버시 토글 | `eye` / `eye.slash` (켜짐 상태) |
| 홈 헤더 우 | 설정 진입 | `gearshape` |
| 홈 footer 우 | 자산 관리 진입 | `square.and.pencil` |
| 공통 헤더 좌 | 뒤로 가기 | `chevron.left` |
| 상세 헤더 우 | 거래 추가 | `plus` |
| 자산 관리 행 | 메뉴바 표시 | `menubar.rectangle` |
| 자산 관리 행 | 숨김 / 삭제 | `eye.slash` / `trash` |
| 설정 | 단축키 등록 | `keyboard` |
| 설정 | 앱 종료 | `power` |
| 검색 | 검색 필드 | `magnifyingglass` |

## 9. 설정 화면 & 다국어

- 진입: 홈 헤더의 ⚙ 아이콘 → 팝오버 내 설정 화면(‹ 로 복귀, 세로 스크롤).
- UI: 섹션 라벨 + **그룹 카드**(rounded), 행 우측에 컨트롤 — 토글 / select
  드롭다운 / 버튼. (OpenUsage 설정 화면 스타일)
- **General**
  - 로그인 시 자동 시작(토글, `SMAppService`)
  - 글로벌 단축키(record 버튼) — 팝오버 열기
  - 메뉴바 로테이션 간격(select: 5초/10초/30초/끔)
- **Appearance**
  - 테마(select: 시스템/Slate/Light, 기본 시스템)
  - 언어(select: 시스템/한국어/English, 기본 시스템)
- **Data**
  - 시세 갱신 주기(select: 1/5/10/15분)
  - 기준 통화(select: KRW/USD/JPY) — 총액·통합 차트 등 **합산 계산에만** 사용
  - 자산 표시 통화(select: **소스 통화 그대로**(기본) / 기준 통화로 환산)
    — 소스 통화면 BTC는 $, 삼성전자는 ₩로 각자 표시
  - 데이터 백업/복원(버튼) · 캔들 캐시 비우기
- **Updates**
  - 자동 업데이트 확인(토글) · 지금 확인(버튼) · 릴리즈 노트 링크
- 맨 아래: **Weave 종료** 버튼(+ ⌘Q) — 메뉴바 앱이라 Dock 종료 동선이 없음.
- Footer: 버전 + 다음 갱신 카운트다운.
- **다국어**: 한국어/영어 2개, String Catalog(`.xcstrings`) 기반. 기본은 시스템
  로케일, 설정에서 앱 단위 오버라이드. 숫자·통화·날짜 포맷은 로케일 규칙
  (`FormatStyle`)을 따름.

## 10. 앱 업데이트 & 버전관리

- **Sparkle 2** 채택 (메뉴바 앱 표준, OpenUsage와 동일 방식).
  - GitHub Releases에 notarized zip 업로드 + `appcast.xml` 게시(GitHub Pages).
  - repo는 **public** — Releases 배포/appcast 호스팅 단순화.
  - EdDSA 서명, 자동 체크 주기 1일, 다운로드→재시작 원클릭.
- 버저닝: semver(`0.x.y`), 릴리즈마다 CHANGELOG. footer에 현재 버전 상시 표시.
- 요구사항: Developer ID 서명 + notarization (없으면 Gatekeeper가 업데이트 차단).

## 11. 데이터 저장

- `Application Support/Weave/portfolio.json` — 자산 · 거래(매수/매도) · 설정.
- 스키마에 `version` 필드, 로드 시 마이그레이션 체인.
- 시세/캔들/환율 캐시는 별도 파일(포트폴리오 데이터와 분리, 삭제해도 무해).

## 12. 구현 순서 (마일스톤)

| # | 범위 | 완료 기준 |
