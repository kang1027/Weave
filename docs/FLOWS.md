# Weave 동작 플로우 (QA/리뷰 기준)

`docs/FEATURES.md`의 명세를 실제 사용자 동작 단위로 풀어놓은 전수 목록.
각 플로우는 "조작 → 기대 동작" 형식이며, 리뷰/QA는 이 문서를 기준으로 검증한다.

## F1. 앱 시작

- **F1.1 최초 실행**: `portfolio.json` 없음 → 빈 문서로 시작. 메뉴바 `Weave`,
  팝오버는 온보딩(검색바 + 안내 + Manual Asset 링크).
- **F1.2 재실행**: 문서 로드 → 시세 폴링·메뉴바 로테이션 시작, 팝오버 첫 열림 시
  1회만 백그라운드 시작(이후 열림은 60초 초과 시에만 즉시 갱신).
- **F1.3 손상/미래 버전 문서**: 로드 실패 시 빈 문서 폴백(로그 남김).
  스키마 version이 앱보다 높으면 로드 거부.
- **F1.4 Dock 아이콘 없음**: LSUIElement/accessory 정책. ⌘Q(설정 화면) 또는
  설정의 종료 버튼으로만 종료.

## F2. 자산 검색·추가·관리

- **F2.1 라이브서치**: 검색창 2자 이상 + 300ms 디바운스 → Binance(로컬 카탈로그
  필터)·Naver(자동완성)·Yahoo(검색 API) 병렬 → 병합 리스트.
  각 행: 로고(크립토=CDN, 그 외 이니셜) · 이름 · 심볼 · 마켓 뱃지(크립토/국장/미장/일장/기타).
- **F2.2 중복 병합**: 국장은 네이버 우선(네이버 응답 있으면 야후 국장 결과 제거),
  크립토는 Binance 우선(같은 베이스 심볼의 야후 크립토 제거). 정확 일치 → 접두 → 포함 순 정렬.
- **F2.3 추가**: 결과 행 클릭 → provider/symbol/통화 자동 세팅으로 자산 생성 →
  저장 → 해당 자산 상세로 이동. 야후는 첫 quote로 통화 확정.
- **F2.4 중복 추가 방지**: 같은 provider+symbol 자산이 있으면 새로 만들지 않고 기존 상세로 이동.
- **F2.5 30개 제한**: 초과 시 추가 UI 비활성 + 안내 문구.
- **F2.6 Manual Asset**: 이름/통화/평가액/차트 포함 여부 입력 → 생성(시세 갱신 없음,
  메뉴바 로테이션 제외). 관리 화면으로 복귀.
- **F2.7 자산 관리 행**: 메뉴바 표시 토글 · 색상 변경(8색) · 숨김/해제 · 삭제.
  삭제는 확인 다이얼로그("이름 + 거래 N건 함께 삭제") 후 거래까지 통째 삭제.
- **F2.8 자산 색상**: 추가 시 팔레트 8색 중 사용 빈도가 가장 낮은 색 자동 할당.
- **F2.9 숨김 자산**: 홈 리스트·링·차트·메뉴바·합산에서 제외. 관리 화면에서만 보임.

## F3. 홈

- **F3.1 링 게이지**: Day(±2%=풀링)·Return(±25%=풀링, 중앙 크게)·Assets(도넛).
  Day/Return은 포트폴리오 손익과 같은 부호 종목만 기여 비율로 분할(기여 큰 순),
  Assets는 비중 상위 4 + "기타"(회색). 세그먼트는 갭 없는 butt cap, 12시 시작 시계방향.
  hover 시 세그먼트 굵어지고 기여액/비중 툴팁.
- **F3.2 총액**: 기준통화 환산 총 평가금 + 일간 등락 솔리드 배지(▲/▼).
- **F3.3 통합 차트**: 일별 Σ(보유수량×종가×환율) 라인+그라디언트.
  표시 구간 = max(기간 시작, 첫 매수일). 매수 이벤트 위치에 자산 로고 원형 마커 —
  hover 시 툴팁(수량@단가·날짜·현재 대비%) + 세로 가이드, 클릭 시 해당 자산 상세로.
- **F3.4 자산별 차트**: 종목별 정규화 %(구간 시작=0%) 멀티라인(자산색). 마커 숨김.
- **F3.5 기간**: 1M/3M/6M/1Y — 변경 시 재계산(캔들은 일봉 캐시 재사용).
- **F3.6 자산 리스트**: 평가금 내림차순 고정. 행: 로고·이름·수량/매수횟수·현재가·등락 배지.
  현재가는 표시 통화 설정(소스/기준환산)을 따름. 클릭→상세.
  컨텍스트 메뉴: 핀 지정/해제 · 숨김 · 삭제. 시세 실패 자산은 stale 아이콘.
- **F3.7 Footer**: `Weave {버전} · Next refresh in {남은시간}` 실시간 카운트다운.
  업데이트 발견 시 "{현재} → {신규} available" 링크로 전환. 우측 ✎ → 자산 관리.
- **F3.8 프라이버시 모드**: eye 토글 → 금액(총액·현재가·손익액·툴팁 금액) 마스킹,
  등락률(%)은 유지. 메뉴바에도 즉시 반영. 앱 재시작 후에도 유지(설정 저장).
- **F3.9 수동 새로고침**: ↻ 클릭 → 시세+환율 즉시 갱신, 카운트다운 리셋.

## F4. 메뉴바

- **F4.1 로테이션**: 표시 대상 = 숨김 아님 + 메뉴바 표시 켬 + manual 아님.
  설정 간격(5/10/30초, 끔)으로 순환. 핀 자산이 있으면 로테이션 없이 핀만 표시.
- **F4.2 형식**: 풀(`BTC $60,000 ▲1.23%`) / 컴팩트(`BTC ▲1.23%`) / 가격만(`$60,000`).
- **F4.3 대상 없음**: 포트폴리오 총액(`₩12,345,678 ▲1.23%`), 총액도 없으면 `Weave`.
- **F4.4 프라이버시**: 금액 `•••••` 마스킹, 등락률 유지. 가격만 형식은 심볼+등락률로 대체.
- **F4.5 시세 실패**: 마지막 성공값 유지(스냅샷), 행에 stale 표시.

## F5. 자산 상세

- **F5.1 헤더**: ‹ 뒤로 · 자산명 · ＋(거래 추가). 현재가 + 등락 배지 + 평단 대비 배지(보유 시).
- **F5.2 차트**: 실제 캔들 종가 라인+자산색 그라디언트, 우측 y축(축약 가격), 하단 x축.
  평단 점선(RuleMark+라벨, 보유 시만). 매수(B, 초록)/매도(S, 빨강) 마커 — 실제 체결가
  좌표, hover 툴팁. 빈 영역 hover 시 크로스헤어(세로+가로 가이드) + 날짜·종가 툴팁.
- **F5.3 기간**: 1M/3M/6M/1Y(일봉) / ALL(주봉, 데이터 5년 초과 시 월봉).
- **F5.4 Trades**: 최신순. 매수 행 우측=현재 대비 %, 매도 행 우측=실현손익.
  상단에 실현손익 누계(이 화면에만 노출). 행 컨텍스트 메뉴: 수정/삭제(확인 다이얼로그).
- **F5.5 Footer**: `N buys · M sells` + `+ Add Trade`.
- **F5.6 자산 삭제 직후**: 상세 화면 잔상 없이 홈/관리로 복귀.
- **F5.7 Manual 자산 상세**: 차트/거래 없음 — 안내 + 관리 화면 링크.

## F6. 거래 입력

- **F6.1 자동 계산**: 수량/단가/총액 중 2개 입력 시 나머지 자동 계산.
  사용자가 만진 필드는 덮어쓰지 않음.
- **F6.2 종가 프리필**: 과거 날짜 선택 시 그날 종가를 단가에 프리필(캔들 캐시, 수정 가능).
  거래 "수정" 진입 시 원래 체결일에 대해선 프리필하지 않음(체결가 보존).
- **F6.3 매도 검증**: 해당 날짜 기준 보유 수량 초과 불가(수정 시 자기 자신 제외).
  가능 수량 표시 + 초과 시 에러 + 저장 비활성.
- **F6.4 미래 날짜 불가**: DatePicker가 오늘까지로 제한.
- **F6.5 저장/수정/삭제 반영**: 평단(이동평균)·실현손익·수량·홈 차트 즉시 재계산.
  실현손익 = (매도단가 − 매도 시점 평단) × 수량, 매도해도 평단 유지.

## F7. 데이터 파이프라인

- **F7.1 시세 폴링**: 설정 주기(1/5/10/15분, 60~900초 클램프)로 전 자산 병렬 갱신.
  일간 변동은 소스 제공값(크립토 24h 롤링, 주식 전일비). 실패 자산은 이전 값 유지+stale.
- **F7.2 캔들 캐시**: `Application Support/Weave/cache/`에 provider·심볼·주기별 파일.
  같은 날 재요청은 캐시 히트(네트워크 0회), 다음날 갱신, 갱신 실패 시 stale 캐시 폴백.
  동시 요청은 in-flight 합류.
- **F7.3 환율**: 현물 1시간 캐시(합산·메뉴바), 일별 시계열은 캔들 캐시 공유(통합 차트).
  같은 통화면 환산 생략(rate=1).
- **F7.4 저장**: 모든 변이 직후 `portfolio.json` 원자적 저장. version 필드 + 마이그레이션 체인.
- **F7.5 Binance 심볼 카탈로그**: exchangeInfo(USDT 페어) 하루 캐시, 실패 시 만료 캐시 사용.

## F8. 설정

- **F8.1 General**: 자동 시작(SMAppService, 번들 실행 시만) · 글로벌 단축키(레코딩,
  ⎋ 취소, 컨텍스트 메뉴로 제거, 트리거 시 팝오버 토글) · 로테이션 간격 · 메뉴바 형식.
- **F8.2 Appearance**: 테마(시스템/Slate/Light — 즉시 전환) · 언어(시스템/한국어/English
  — 즉시 전환, 숫자·날짜 포맷 로케일 연동).
- **F8.3 Data**: 갱신 주기(폴링 루프 재시작) · 기준 통화(KRW/USD/JPY — 총액·링·차트
  즉시 재계산) · 자산 표시 통화(소스 그대로/기준 환산) · 백업(JSON 내보내기) ·
  복원(검증 실패 시 에러, 성공 시 전체 교체+재갱신) · 캔들 캐시 비우기.
- **F8.4 Updates**: 자동 업데이트 토글 · 지금 확인(번들+피드 있을 때만 활성).
- **F8.5 종료**: Weave 종료 버튼 + ⌘Q.
- **F8.6 설정 영속**: 모든 설정 변경은 즉시 저장, 재시작 후 유지.

## F9. 업데이트·배포

- **F9.1 개발 실행**: `swift run Weave` — Sparkle/자동시작 비활성(조용히), 나머지 전 기능 동작.
- **F9.2 번들**: `scripts/bundle.sh` → `dist/Weave.app`(Info.plist: LSUIElement,
  SUFeedURL, 버전 주입 + Sparkle.framework + 리소스 번들).
- **F9.3 릴리즈**: `scripts/release.sh` → ED 공개키 가드 → 서명 → 공증 → zip →
  EdDSA 서명까지 자동. GitHub Release 업로드와 `docs/appcast.xml` 항목 추가는
  스크립트가 안내하는 수동 단계(GitHub Pages 서빙).
- **F9.4 업데이트 수신**: 번들 앱이 매일 appcast 확인 → 발견 시 footer 뱃지 → 클릭 설치.
